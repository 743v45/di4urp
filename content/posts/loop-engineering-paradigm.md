## 二、什么是"循环"？

如果你做过 Agentic Engineering 或 Vibe Coding，你一定熟悉这个流程：

```
你提示代理 → 代理写代码 → 你等待 → 你再提示 → 重复
```

**循环改变了这一切。**

你不再亲自告诉代理该做什么，而是**设计循环**——本质上就是指定一个终态目标。你把目标交给代理，代理不仅会自动启动，还会持续运行，直到目标达成。

### 循环的两大要素

| 要素 | 说明 |
|------|------|
| **触发器（Trigger）** | 循环何时启动 |
| **目标（Goal）** | 循环何时终止（必须可验证） |

目标的验证方式可以是：
- **确定性验证**：测试全部通过、函数正确执行、无错误。
- **非确定性验证**：由另一个 LLM/代理判断"目标是否已达成"。

这与强化学习（RL）如出一辙：需要一个可验证的奖励信号，让 AI 知道何时成功。

## 三、实际演示

### 在 Cursor 中的 Automations

在 Cursor 的 **Automations** 标签页中，可以设置自动化循环。例如：

> **触发器：** 每次在 Astro Hub 项目中打开 PR  
> **指令：** 审查 PR，查找潜在问题，自动修复并提交回同一 PR。确保所有测试通过；如果没有通过则修复。确保所有 CI 绿灯。

这就是一个完整的循环：触发器 + 目标。

### 三种触发器类型

| 类型 | 示例 |
|------|------|
| **事件触发** | PR 打开、文件变更等 |
| **定时触发（Cron）** | 每 30 分钟、每小时、每天、每周 |
| **人工触发** | 人类手动启动 |

### 在 Claude Code 中使用 `/loop`

Claude Code 提供了内置的 `/loop` 命令：

```
/loop 5 minutes "对比我们已构建的内容与完整规格说明（spec.md），继续构建直到完成全部规格"
```

每隔 5 分钟启动一个代理，代理判断还有哪些内容待构建，然后开始构建，循环往复直到目标达成。

也可以去掉时间间隔，由人类一次性启动：

```
/loop "继续循环直到达成最终目标"
```

循环可以更加复杂——可以为循环赋予技能（skills），甚至编写代码让循环在迭代中变得更聪明。但**最基本的循环定义始终是：触发器 + 目标**。

## 四、循环 vs 自动化

这是一个关键的区分：

| 维度 | 自动化（Automation） | 循环（Loop） |
|------|---------------------|-------------|
| 决策 | 无，执行预设步骤 | **有，循环内部包含决策** |
| 终止条件 | 执行完毕后结束 | 代理判断目标是否达成 |
| 本质 | 执行一系列提示/代码 | 代理自主决定何时终止 |

**核心区别：** 循环赋予了代理判断"是否已达到目标"的能力，而不仅仅是执行一系列预设指令。

## 五、循环工程的争议

### 争议一：设置困难

基础循环确实简单。但如果你要构建一个能无限构建完整产品的"代码工厂"，那就非常困难了。

对于没有确定性可验证目标的任务（如"构建这个功能"），如何定义终态？你需要预先写出完整规格。而对许多人来说，构建功能的过程本身就是探索的过程——哪些需要、哪些不需要，需要不断迭代。而循环工程要求你将这个过程完全交给代理，然后走开。

### 争议二：成本高昂

```
抽象程度越高 → 人类参与度越低 → token 消耗越多 → 账单越贵
```

Peter Steinberger 曾透露他每月 token 用量高达 **130 万美元**。只有 OpenAI、Anthropic 等极少数公司能给员工提供无限 token 预算。

因此，**循环工程目前确实不属于大多数人**。

### 技术发展的必然规律

历史上每一次技术扩散，生产效率都在提升，价格都在下降。今天的昂贵就是明天的便宜。但当下，循环工程确实是奢侈的。

## 六、工程师群体的巨大分化

目前，工程领域正在出现严重的分化：

- **前 0.01%**：使用循环工程等新技术，拥有无限或高额 token 预算。
- **其余 99.99%**：在传统方式下工作，公司正在削减 AI 开支。

---

## 七、未来的终极思考

### 人类会永远留在循环中吗？

目前，人类仍然在循环中——因为我们仍在决定目标和方向。

但未来可能出现这样一个世界：

> **AI 拥有品味。** AI 能够自主决定构建哪些功能、哪些产品、哪些公司。AI 能够设计它自己的工厂。

这正是**递归自我改进（Recursive Self-Improvement）** 的起点——AI 不再等待人类设定目标，而是自己设定目标。Anthropic 刚刚发布了一篇关于递归自我改进的完整长文。

---

## 八、总结

| 要点 | 内容 |
|------|------|
| **核心概念** | 循环 = 触发器 + 可验证目标 |
| **三种触发器** | 事件触发、定时触发、人工触发 |
| **关键区分** | 循环包含决策，自动化不包含 |
| **当前局限** | 设置困难、成本高昂 |
| **未来方向** | 软件工厂 → 递归自我改进 → AI 自主设定目标 |

> **循环工程不是适合每个人的工具，但绝对是工程学的未来。** 我们会继续构建"构建软件的软件工厂"。作为工程师，我们将逐渐、然后突然不再向代理发送提示词来写代码——我们将设计工厂，让代理自主运行。
