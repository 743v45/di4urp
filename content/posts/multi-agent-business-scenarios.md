## 引言

多 Agent 系统不是学术概念——它已经可以落地到真实的商业场景中。以 [OpenClaw](https://github.com/openclaw/openclaw)（37 万 star 的开源个人 AI 助手平台）为例，它内置了完整的多 Agent 路由能力：

- **Multi-Agent Routing**：按渠道/账户/对话方路由到隔离 Agent
- **Session Tools**：Agent 之间通过 `sessions_send` 互相通信
- **Sub-Agent**：用 `sessions_spawn` 创建后台子 Agent
- **Delegate Architecture**：Agent 以独立身份代表组织行事
- **Parallel Specialist Lanes**：并行专家通道，避免资源争抢

基于这套能力，我整理了 **12 个行业、30 个场景**，每个都可以用纯配置实现，不需要写代码。

## 金融 / 投资

### 场景 1：投研三人组

三个专家 Agent 并行分析，最后汇总成投资决策建议。

```
┌─ 「宏观 Agent」：跟踪政策、利率、经济数据
├─ 「行业 Agent」：分析特定赛道（AI、新能源、医药）
└─ 「标的 Agent」：个股财务分析、估值建模
```

**演示流程**：用户问 "该不该买入 XXX" → 三个 Agent 并行输出各自视角 → 汇总成综合决策建议。

### 场景 2：风控双保险

交易 Agent 发起操作，风控 Agent 二次检查。

```
交易 Agent 发起交易
  ↓ sessions_send
风控 Agent 检查（额度、合规、黑名单）
  ↓ 通过 → 交易 Agent 执行
  ↓ 拦截 → 通知人工审核
```

**演示流程**：展示一笔交易被风控 Agent 拦截的场景，突出二次验证和自动升级机制。

## 医疗 / 健康

### 场景 3：问诊分诊台

根据症状描述自动路由到专科 Agent。

```
用户描述症状
  ↓
分诊 Agent（判断科室）
  ├─ 呼吸科 → 呼吸 Agent
  ├─ 皮肤科 → 皮肤 Agent
  └─ 紧急 → 立即建议就医 + 发送警报
```

**演示流程**：描述不同症状 → 路由到不同专科 Agent，每个有独立的医学知识库和话术风格。

### 场景 4：健康管理三人组

饮食、运动、睡眠三个维度并行分析。

```
┌─ 「饮食 Agent」：分析卡路里、营养配比
├─ 「运动 Agent」：推荐训练计划
└─ 「睡眠 Agent」：分析睡眠质量、给出改善建议
```

**演示流程**：用户发一张饮食照片 → 三个 Agent 并行分析 → 生成综合健康报告。

### 场景 6：语言学习伙伴

```
┌─ 「外教 Agent」：纯英语对话，纠正语法
├─ 「语法 Agent」：解释规则、举例
└─ 「词汇 Agent」：生成单词卡、记忆曲线提醒
```

**演示流程**：用户用英语聊天 → 外教 Agent 纠正 → 语法 Agent 详细讲解 → 词汇 Agent 收集生词。

报修 Agent → 记录问题、派单

## 创意 / 设计

### 场景 15：创意脑暴会

发散 → 收敛 → 细化三阶段。

```
┌─ 「发散 Agent」：狂出创意点子（不限数量和质量）
├─ 「收敛 Agent」：筛选可行性、打分排序
└─ 「细化 Agent」：将选中方案完善成可执行方案
```

**演示流程**：给一个主题（如"奶茶店新品"）→ 三个 Agent 输出完整创意流程。

### 场景 16：多版本测试组

同一个需求，三个风格同时产出，投票 Agent 选最佳。

```
同一个需求
  ↓
版本 A Agent（保守稳妥）
版本 B Agent（前卫大胆）
版本 C Agent（平衡型）
  ↓
投票 Agent 评估 → 推荐最佳版本
```

**演示流程**：一个文案需求 → 三个版本同时产出 → 投票 Agent 给出推荐和理由。

## 企业内部

### 场景 23：合同审阅三人组

法务、财务、业务并行审阅。

```
┌─ 「法务 Agent」：查法律风险、合规问题
├─ 「财务 Agent」：查付款条款、税务风险
└─ 「业务 Agent」：查商业条款、交期承诺
```

**演示流程**：上传合同文本 → 三个 Agent 并行审阅 → 汇总风险点和修改建议。

### 场景 24：IT 运维组

监控 → 自动修复 / 升级决策。

```
监控 Agent 检测异常
  ├─ 已知问题 → 自动化 Agent 直接修复
  └─ 未知问题 → 升级给人工 + 记录案例
```

**演示流程**：模拟系统故障 → 展示自动修复和升级决策。

### 场景 25：品牌舆情监控

定时巡逻 + 分级响应。

```
监控 Agent（Cron 定时巡逻）
  ├─ 绿色：普通提及 → 记录归档，不打扰
  ├─ 黄色：疑似差评 → 推送给运营 Agent 起草回复话术
  └─ 红色：公关危机 → 推送给 PR Agent + 立即通知负责人
```

**演示流程**：设定 Cron Job → 模拟负面评论 → 监控 Agent 判断级别 → sessions_send 给对应 Agent。

### 场景 26：内容编辑部

写手 → 编辑 → 排版流水线。

```
写手 Agent 写初稿
  ↓ sessions_send
编辑 Agent 审稿（查事实、改措辞、打分）
  ↓ 分数 < 80 → 写手 Agent 修改
  ↓ 分数 ≥ 80 → 排版 Agent 生成最终版
```

**演示流程**：给主题 → 写手出初稿 → 编辑审稿 → 打分循环 → 排版输出。

### 场景 27：多语言全球客服

按语言自动路由。

```
WhatsApp 消息进来
  ├─ 中文 → 中文 Agent（小美）
  ├─ English → English Agent (Alex)
  └─ 日本語 → 日本語 Agent（田中さん）
```

**演示流程**：三个手机/三个号，各发不同语言 → 各自路由到对应语言 Agent → 展示本地化服务。

### 场景 28：代班秘书（Delegate）

Agent 有独立身份，代表组织行事。

```
CEO："帮我安排下周所有 1on1"
  ↓
Delegate Agent（用 CEO 的日历权限）
  → 查看所有下属空闲时间
  → 创建会议邀请（发件人显示 "助理 on behalf of CEO"）
  → 完成后汇报结果
```

**演示流程**：展示 Delegate 的 SOUL.md（硬性规则）→ 发一个日历操作指令 → 展示 "代表 xxx 发送" 效果。

### 场景 29：多渠道智能客服

按渠道自动路由到不同专业 Agent。

```
客户 WhatsApp 留言 → 销售顾问 Agent（擅长报价、产品推荐）
客户 Telegram 投诉 → 售后支持 Agent（擅长安抚、工单处理）
内部 Slack 询问 → 内部知识库 Agent（擅长政策查询、流程指引）
```

**演示流程**：三个渠道发不同类型消息 → 各自路由到对应 Agent → 展示三人格、三套工具权限的隔离。

### 场景 30：电商运营三人组

选品 → 文案 → 投放完整闭环。

```
┌─ 「选品 Agent」：监控趋势、推荐爆款候选
├─ 「文案 Agent」：写商品标题、详情页、小红书种草文案
└─ 「投放 Agent」：生成广告素材 brief、排期建议
```

**演示流程**：运营在群聊说"帮我看看最近什么好卖"→ 选品 Agent 返回候选 → 选一个 → 文案 Agent 产出 → 投放 Agent 出计划。

## 技术底座：OpenClaw 多 Agent 能力

这些场景的技术基础是 OpenClaw 的四层多 Agent 架构：

| 能力 | 说明 | 对应场景 |
|------|------|----------|
| **Multi-Agent Routing** | 按渠道/账户/对话方路由到隔离 Agent | 分诊台、服务台、多语言客服 |
| **Session Tools** | Agent 间通过 `sessions_send` 通信 | 风控双保险、编辑部、舆情监控 |
| **Sub-Agent** | `sessions_spawn` 创建后台子 Agent | 投研、合同审阅、运维 |
| **Delegate** | Agent 以独立身份代表组织行事 | 代班秘书 |
| **Parallel Lanes** | 并行专家通道避免资源争抢 | 门店督导、健康管理 |

所有场景都可以通过配置 `openclaw.json` 中的 `agents`、`bindings`、`channels` 实现，不需要写代码。
